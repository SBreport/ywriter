# youbox 로드맵 & 전략 문서

> 이 문서는 세션 간 컨텍스트 연속성을 위한 **의사결정 기록 + 현재 상태 + 다음 할 일** 을 담고 있습니다. 새 세션 시작 시 이 문서를 먼저 읽고 작업을 이어받으세요.

최종 업데이트: 2026-04-24

## 🎯 북극성 (North Star)

> **youbox = 유튜브 창작자의 기획 의도 → 촬영 → 편집 → 발행 → 회고 전 과정을 기록하여,  
> 영상이 쌓일수록 창작 품질이 지속 상승하는 SaaS 제품**

### 핵심 가치 명제
1. **맥락 보존** — 기획 의도가 촬영자/편집자에게 100% 전달
2. **학습 축적** — 영상이 쌓일수록 다음 영상이 나아짐
3. **반복 자산화** — 자주 쓰는 스타일/체크리스트가 프리셋화
4. **협업** — 솔로부터 외주 팀까지 비동기/동기 소통

### 차별화 논리 (왜 노션/Obsidian이 아닌가)
- 노션/Obsidian = 자유도 높지만 "구조 강제 없음" → 유저가 매번 설계
- Frame.io = 편집 협업만 특화 → 기획-원고 부재
- **youbox = 유튜브 워크플로우에 특화된 강제 구조** — 이게 경쟁력

## 📊 현재 상태 (2026-04-24 기준)

### 구현된 기능

**아키텍처:**
- 순수 HTML/CSS/JS 정적 사이트
- GitHub Pages 호스팅: https://sbreport.github.io/youbox/
- GitHub 레포: https://github.com/SBreport/youbox
- 데이터: localStorage + IndexedDB (이미지)
- ZIP 백업/복원 (JSZip CDN)

**페이지 구조:**
- `index.html` — 홈 (프로젝트 리스트)
- `project.html` — 프로젝트 쉘 (4개 챕터 탭)
- `warehouse.html` — 썸네일 창고 (글로벌)

**JS 모듈:**
- `projects-db.js` — 프로젝트 CRUD + IndexedDB 이미지
- `folders-db.js` — 창고 폴더 트리 (2단)
- `zip.js` — ZIP 백업/복원 (프로젝트 단위, 전체, 창고)
- `youtube-api.js` — YouTube Data API 래퍼
- `home.js` — 홈 페이지
- `project.js` — 프로젝트 쉘 (탭 전환, 컨텍스트 바)
- `chapter1-brief.js` — 기획 (썸네일/제목/독자/목적)
- `chapter2-script.js` — 원고 (벤치마킹 + 내 원고 + BPT 템플릿)
- `chapter3-broll.js` — B-roll (대본 읽기 전용 + 태그/메모)
- `chapter4-edit.js` — 편집 계획 (브리프 + 섹션별 태그) ⚠️ 재설계 필요
- `warehouse.js` — 썸네일 창고

**챕터별 구현 상태:**

| 챕터 | 상태 | 주요 기능 |
|------|------|----------|
| Ch 1 기획 | ✅ 완성도 높음 | 썸네일 수집(붙여넣기/드래그/URL), 영상 제목/썸네일 타이틀/독자/목적, YouTube API 자동 메타 |
| Ch 2 원고 | ✅ 완성도 높음 | 벤치마킹 참고, 3가지 시작 방법(BPT/빈/업로드), H1-H3 계층, 훅 자동 강조, 벤치마킹 접기, 레이아웃 개선(내 원고 중앙) |
| Ch 3 B-roll | ✅ 완성도 높음 | 섹션별 태그(화면녹화/별도녹화/자료화면) + 메모, 촬영 체크리스트 |
| Ch 4 편집 | ⚠️ **재설계 필요** | 현재: 얕은 브리프 + 9개 태그 + 메모. 실무 편집 브리프 수준 아님 |
| Ch 5 발행 | ❌ 미구현 | 필요 |
| Ch 6 회고 | ❌ 미구현 | 필요 |
| Ch 0 채널 프리셋 | ❌ 미구현 | 필요 |

**부가 기능:**

- ✅ 30초 자동 저장 + Ctrl+S

- ✅ 창고 직접 URL 수집 (프로젝트 무관)

## 🧠 전략 논의 기록 (2026-04-24)

### 계기
사용자(@SBreport)가 현재 Ch 4가 "산발적이고 안 쓰이는 기능이 섞여 있다"고 지적.  
논의 결과 Ch 4의 문제는 구조보다 **youbox 전체 정체성**의 문제였음.

### 주요 반전 포인트
1. **사용자가 밝힌 진짜 목표: SaaS 판매**
   - 현재 로컬 정적 사이트 한계에 기능을 구속하지 말 것
   - 본인 dogfooding + 다른 유튜버들에게 판매

2. **Chapter 4 "기획 vs 편집" 혼재 지적**
   - "10초 안에 후킹" 같은 건 기획 단계 결정 사항
   - 편집 단계는 "이미 기획된 걸 실행하는" 단계
   - 현재 Ch 4는 이 둘을 혼동

3. **편집 챕터를 "제거"로 해결하려 했던 Claude의 실수**
   - 얕은 기능을 지우는 대신 제대로 두텁게 만드는 게 맞음
   - 외주 편집 시나리오는 진짜 가치 영역

### 최종 합의된 정체성
**"제작 맥락 허브" (Context Hub for Creators)**
- 솔로 유튜버에겐 "과거 나 ↔ 현재 나" 협업
- 팀 유튜버에겐 "기획자 ↔ 촬영자 ↔ 편집자" 허브
- 데이터가 쌓일수록 가치 증가하는 구조

## 🔬 Chapter 4 재설계 세부안 (다음 세션 시작 지점)

### 문제 의식
- 현재: 얇은 브리프 + 9개 태그 + 섹션별 메모 — 산발적
- 기획과 편집 단계가 뒤섞임
- 실무 편집 브리프 수준 아님
- 외주 편집자가 이걸 받고 "바로 작업 시작"할 수 있는 수준이 아님

### 재설계 원칙
1. **실무 편집 브리프 표준 반영**
2. **기획 관련 내용은 컨텍스트 바로 표시 (중복 입력 X)**
3. **편집 실행 단계 고유 정보에 집중**
4. **협업 허브 기능** (링크/버전/피드백)
5. **채널 프리셋 분리** (Ch 0와 연결)
